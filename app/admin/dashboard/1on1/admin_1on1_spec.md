# 📄 [SPEC] 모바일 최적화 Admin 매칭 마스터 보드 (V5.4)

---

### 1. 설계 개요
* **목적**: 사회자(MC)가 현장에서 모바일 기기로 중복 없는 최적의 1:1 매칭을 확정하고 가이드를 제공함.
* **핵심 원칙**: 
    1. **여성 우선 순위**: 여성 참가자 전원에게 중복되지 않는 최적의 남성 1순위를 배정.
    2. **상호 배타성**: 특정 남성이 한 여성의 1순위 확정 시, 타 여성의 후보군에서 즉시 제외 ($1:1$ 완전 매핑).
    3. **모바일 퍼스트**: 한 손 조작이 가능한 카드형 UI 및 하단 시트(Bottom Sheet) 인터페이스 적용.

---

### 2. 매칭 알고리즘 (Exclusivity Logic)
단순 합산 점수가 아닌 '전역 최적화'를 위해 아래 로직을 수행합니다.

#### 2.1. 배타적 매칭 프로세스
1. **점수 산출**: 가치관($70\%$) + 피드($30\%$) 합산 데이터를 전수 연산.
2. **1순위 경합 해결**: 
   - 남성 A가 여성 1, 2에게 중복 1순위일 경우, `[점수 + 맞하트 여부]`가 높은 커플을 우선 확정.
   - 밀려난 여성에게는 남은 남성 리스트 중 차상위 점수자를 1순위로 자동 재배정.
3. **최종 확정**: Gale-Shapley 알고리즘 변형을 통해 여성 기준 모든 매칭의 상호 배타성 확보.

### 4. 안내 멘트 가이드 (Expanded View)
버튼 클릭 시 아래에서 위로 슬라이드되는 하단 시트 내 구성안입니다. 사회자가 리딩하기 좋게 **큰 폰트**를 사용합니다.

#### 4.1. 상세 가이드 구성
* **🔥 커플 매칭 포인트**
    - 핵심 가치관: **[자유]** (두 분 다 3회차 이상 입찰한 '진심' 커플)
    - 취향 일치도: 피드 하트 **5개** 일치 (시각적 호감 최고치)

* **🎤 사회자 오프닝 멘트**
    - "{여성닉네임}님, {남성닉네임}님! 두 분은 **[{가치관명}]** 가치관에 3회차 이상 비딩하신 '진심' 커플입니다. 특히 두 분 다 {가치관 특징}을 중시하시네요!"

* **💬 강력 추천 첫 대화 (Icebreaker)**
    - "1on1 리포트에 추천되어 있듯이, 첫 대화로 **'{추천 질문}'**을 가볍게 얘기 나눠보시는 건 어떨까요? 서로의 온도 차이를 확인해보는 재미가 있을 거예요! 😊"

### 5. 기술 요구사항
* **Responsive**: 모바일 브라우저 최적화 (뷰포트 설정 및 터치 타겟 $44 \times 44$px 확보).
* **Manual Override**: 사회자가 매칭 수정 시, 실시간으로 '남성 중복 매칭' 여부를 체크하여 경고 알림 노출.
* **Sync**: n8n 매칭 연산 완료 즉시 `final_reports` 테이블에서 데이터를 로드하여 화면 갱신.
